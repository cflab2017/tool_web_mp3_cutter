# 오디오 자르기 — 사용법

원하는 구간만 잘라 무손실 WAV로 저장하는 단일 HTML 도구입니다.
**파일은 서버로 전송되지 않으며, 모든 처리는 브라우저 안에서** 이루어집니다. 외부 라이브러리도 사용하지 않습니다.

![오디오 자르기 화면](images/screenshot-desktop.png)

---

## 30초 요약

1. `index.html`을 브라우저로 엽니다.
2. 오디오 파일을 **끌어다 놓거나** 클릭해서 선택합니다.
3. 파형에서 **시작·끝 핸들을 드래그**해 자를 구간을 정합니다.
4. **자르고 다운로드**를 누르면 선택 구간이 `WAV`로 저장됩니다.

---

## 자세한 사용법

### 1. 파일 불러오기
- 점선 영역에 오디오 파일을 끌어다 놓거나, 영역을 클릭해 파일 선택창을 엽니다.
- `MP3 · WAV · M4A · OGG · FLAC` 등 브라우저가 디코딩할 수 있는 형식을 지원합니다.
- 불러오면 파형과 함께 파일명·길이·샘플레이트·채널 정보가 표시됩니다.

### 2. 구간 선택하기
구간을 정하는 방법은 세 가지입니다.

- **핸들 드래그**: 파형 양옆의 세로 핸들을 좌우로 끌어 시작·끝을 맞춥니다.
- **새로 그리기**: 파형의 빈 곳을 누른 채로 드래그하면 그 범위가 새 선택 구간이 됩니다.
- **직접 입력**: 오른쪽의 `시작`/`끝` 칸에 시간을 입력합니다. (`분:초.밀리초` 또는 초 단위, 예: `1:23.500` 또는 `83.5`)

선택 구간은 밝은 앰버색으로, 바깥쪽은 어둡게 표시됩니다. `선택 길이`로 결과 길이를 미리 확인할 수 있습니다.

### 3. 미리 듣기
- **재생 / 일시정지**: 현재 재생 위치부터 전체를 재생합니다.
- **구간 재생**: 선택한 구간만 재생해 자르기 전에 확인합니다.
- 파형을 가볍게 **클릭**하면 그 지점으로 재생 위치(플레이헤드)가 이동합니다.
- `재생위치` 버튼을 누르면 현재 플레이헤드 위치를 시작/끝으로 지정합니다 — 들으면서 정확히 자르기 좋습니다.

### 4. 자르고 저장하기
- **자르고 다운로드**를 누르면 선택 구간이 `원본이름_cut.wav`로 다운로드됩니다.
- 출력은 원본의 샘플레이트·채널을 그대로 따르는 **16-bit PCM WAV(무손실)** 입니다.

## 자주 묻는 질문 (FAQ)

**Q. 파일이 서버로 올라가나요?**
아니요. 디코딩·자르기·저장이 모두 브라우저 안에서 처리되며, 네트워크로 전송되지 않습니다. 인터넷 연결 없이도 동작합니다.

**Q. 왜 MP3가 아니라 WAV로 저장되나요?**
브라우저에는 내장 MP3 인코더가 없고, 이 도구는 외부 라이브러리를 쓰지 않는다는 원칙을 지킵니다. 그래서 추가 의존성 없이 항상 동작하는 **무손실 WAV**로 내보냅니다. MP3가 필요하면 저장한 WAV를 별도 변환 도구로 바꾸면 됩니다.

**Q. 어떤 형식을 넣을 수 있나요?**
브라우저가 디코딩할 수 있는 형식이면 됩니다(보통 MP3·WAV·M4A/AAC·OGG·FLAC 등). 디코딩에 실패하면 화면에 안내 메시지가 표시됩니다.

**Q. 긴 파일도 되나요?**
됩니다. 다만 매우 긴 파일은 디코딩과 WAV 생성에 메모리를 더 쓰며, WAV는 압축되지 않으므로 결과 파일이 원본보다 클 수 있습니다.

**Q. 페이드나 볼륨 조절 같은 편집도 되나요?**
이 버전은 **자르기 한 가지**에 집중합니다. 핵심 동작을 흠 없이 제공하는 것이 목표입니다.

**Q. 한 번에 여러 구간을 자를 수 있나요?**
현재는 한 번에 한 구간입니다. 필요하면 구간을 바꿔가며 여러 번 내보내면 됩니다.

## English (summary)

A single-file, in-browser **audio cutter**. Load an audio file, drag the start/end
handles on the waveform to choose a region, preview it, then **Cut & download** to
save the selection as a lossless 16-bit PCM **WAV**.

- No upload — everything runs in your browser, no external libraries.
- Input: any format your browser can decode (MP3, WAV, M4A, OGG, FLAC, …).
- Output is WAV because browsers have no built-in MP3 encoder; convert the WAV
  afterward if you need MP3.
